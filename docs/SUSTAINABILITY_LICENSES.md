### Combining with Traditional Open Source Licenses

Sustainability licenses are designed to work alongside traditional OSS licenses:

**Dual Licensing Structure**:

```
Primary License: MIT License (traditional IP/legal terms)
Additional License: SUL-1.0 (sustainability requirements)
```

**How It Works**:
- Traditional license (MIT, Apache, GPL) governs intellectual property
- Sustainability license adds environmental/social/ethical requirements
- Both must be complied with

**Example License Headers**:

```
SPDX-License-Identifier: MIT AND SUL-1.0
```

or

```

See LICENSE-MIT.md and LICENSE-SUL.md for details.
```

## ğŸš€ Implementation Guide

### Step 1: Choose Your License(s)

Review the six licenses and select the one(s) that align with your project goals and values. Consider:
- Your project's primary purpose and impact
- Computational resource requirements
- Community and governance structure
- Social and ethical implications
- Environmental footprint

### Step 2: Add License Files

Copy the appropriate license file(s) from the [License Library](../licenses/):

```bash
# Single license
cp licenses/SUL-1.0.md YOUR_PROJECT/LICENSE-SUL.md

# Multiple licenses
cp licenses/EAL-1.0.md YOUR_PROJECT/LICENSE-EAL.md
cp licenses/CAL-1.0.md YOUR_PROJECT/LICENSE-CAL.md
```

### Step 3: Update README

Add a license section to your README:

```markdown
## License

This project is licensed under:
- MIT License - for intellectual property terms
- Sustainable Use License (SUL-1.0) - for sustainability requirements

See [LICENSE-MIT.md](LICENSE-MIT.md) and [LICENSE-SUL.md](LICENSE-SUL.md) for details.
```

### Step 4: Update Package Metadata

Add license information to your package files:

**package.json** (Node.js):
```json
{
  "license": "(MIT AND SUL-1.0)",
  "licenses": [
    {
      "type": "MIT",
      "url": "https://opensource.org/licenses/MIT"
    },
    {
      "type": "SUL-1.0",
      "url": "https://github.com/Violet-Site-Systems/.github/blob/main/licenses/SUL-1.0.md"
    }
  ]
}
```

**setup.py** (Python):
```python
setup(
    license="MIT AND SUL-1.0",
    classifiers=[
        "License :: OSI Approved :: MIT License",
        "License :: Other/Proprietary License",
    ]
)
```

**Requirements**:
- Apply principles and best practices
- Make good-faith efforts toward compliance
- Document sustainability considerations

**Exemptions**:
- Formal reporting typically not required
- Third-party verification not needed
- Flexibility in implementation

### Medium Deployments

**Requirements**:
- Implement monitoring and measurement
- Document improvement processes
- Regular internal reviews

- Commercial revenue generation

### Large-Scale Operations

**Requirements**:
- Full formal reporting
- Third-party verification
- Public disclosure of detailed metrics
- Regular audits and assessments
- Science-based targets and commitments

**Typical Threshold**:
- 10,000+ users or
- Enterprise/institutional deployment or
- Major infrastructure services
